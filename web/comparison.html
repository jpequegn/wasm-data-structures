Learning Journey Summary

This project implements 5 fundamental data structures using WebAssembly and JavaScript, demonstrating the trade-offs between different approaches for different problems:

Key Insights

  • Hash Tables (Phases 1-2): Unbeatable for simple O(1) lookup, but lose ordering and prefix capabilities
  • Balanced Trees (Phase 3): Maintain order and balance, enabling range queries and sorted traversal at O(log n)
  • Probabilistic Structures (Phase 4): Skip Lists show that randomization can compete with deterministic balancing
  • Specialized Structures (Phase 5): Tries demonstrate how exploiting problem structure (strings, prefixes) enables specialized operations
  • WASM Integration: All structures benefit from Rust+WASM compilation for 2-10x performance vs pure JavaScript

Design Principles Applied

  • Choose the right tool for the problem (no universal best structure)
  • Profile before optimizing (metrics tracking in every structure)
  • Understand trade-offs (space vs time, simplicity vs guarantees)
  • Leverage language strengths (Rust safety, WASM performance, JS simplicity)
  • Document learning (blog posts explaining each algorithm)
